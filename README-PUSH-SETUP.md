# Налаштування надійних push-сповіщень (TS Розклад v5)

Що змінилось: тепер сповіщення шле не телефон, а маленька безкоштовна функція на
Netlify, яку раз на хвилину "будить" зовнішній безкоштовний сервіс cron-job.org.
Тому нагадування за 10/5 хв до пари приходитимуть, навіть якщо застосунок повністю
закритий і телефон лежить у кишені годинами.

## 1. Задеплой проєкт на Netlify

Завантаж усі файли з цього архіву (включно з папкою `netlify/functions/`) у свій
Netlify-сайт як зазвичай (drag&drop у Netlify Deploys, або через git-репозиторій,
якщо він уже підключений). Netlify сам побачить `netlify.toml` і `package.json`
та встановить `web-push` і `@netlify/blobs` під час білду.

> Якщо сайт зараз на infinityfree.me / іншому хостингу без функцій — саме для push
> потрібен переїзд на Netlify (безкоштовний тариф підходить). Статичні файли
> (index.html, sw.js, іконки) лишаються такими ж, просто хостяться на Netlify.

## 2. Згенеровані ключі VAPID (уже готові, нічого генерувати не треба)

Публічний ключ уже вписаний прямо в `index.html` (константа `VAPID_PUBLIC_KEY`).
Приватний ключ треба додати як **змінну середовища** в Netlify (він не повинен
лежати у коді, який бачать усі):

Netlify → Site configuration → Environment variables → додай:

```
VAPID_PUBLIC_KEY  = BOMVi7PS07KvuuGP7BuS9qGiW-GrdhxBuj5Q-hcDvQQSf-CXoXF7-RidvTr1YAe8jMD3p6ySgv3zqAldFrmSlvQ
VAPID_PRIVATE_KEY = A-UkxlxlbsZ1ByyOojqzdDSGan6scBRoyZtHqJ8xN5o
VAPID_SUBJECT     = mailto:твоя_будь-яка@пошта.com
CRON_SECRET       = придумай-будь-який-довгий-рядок-навмання
```

`CRON_SECRET` — необов'язковий, але бажаний: без нього твою функцію
`check-notifications` теоретично може смикати будь-хто в інтернеті, хто вгадає URL.
З секретом — тільки той, хто знає рядок.

Після додавання змінних — зроби redeploy сайту (Netlify → Deploys → Trigger deploy),
щоб функції їх підхопили.

## 3. Підключи безкоштовний "будильник" на cron-job.org

1. Зареєструйся на https://cron-job.org (безкоштовно, email + пароль).
2. Створи нову задачу (Create cronjob):
   - **URL**: `https://ТВІЙ-САЙТ.netlify.app/.netlify/functions/check-notifications?secret=придумай-будь-який-довгий-рядок-навмання`
     (той самий секрет, що в `CRON_SECRET`)
   - **Schedule**: кожну хвилину (`* * * * *`)
3. Збережи. Готово — тепер щохвилини Netlify перевірятиме твій розклад і сама
   шле push, коли треба.

## 4. Увімкни сповіщення в самому застосунку

Відкрий застосунок → Налаштування → «Активувати сповіщення» (як і раніше).
Тепер це не просто дозвіл браузера — телефон також підпишеться на push і
відправить підписку на сервер. У Налаштуваннях з'явиться індикатор:
🟢 «Push активний» — значить усе працює.

Кожного разу, коли ти міняєш розклад, предмети чи налаштування нагадувань —
застосунок автоматично (через ~1.5 сек) синхронізує їх на сервер, так що
Netlify завжди знає актуальний розклад.

## Обмеження, які варто знати

- Якщо ти повністю видалиш дозвіл на сповіщення в браузері — підписка на
  сервері стане непридатною, і функція сама її прибере після першої невдалої
  спроби відправки.
- Розклад зберігається на сервері окремо від локального (localStorage) —
  вони синхронізуються автоматично, поки застосунок відкритий і сповіщення
  дозволені.
- Безкоштовний план cron-job.org дозволяє мінімальний інтервал 1 хвилина —
  цього достатньо для нагадувань за 10 і 5 хвилин.
